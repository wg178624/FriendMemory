# Friend Memory Project

Friend Memory Project 是一个面向“朋友类 AI”的关系记忆系统实验项目。它关注的问题不是让 AI 记住更多事实，而是让长期陪伴型 AI 以更像人类朋友的方式理解、保存、关联和使用共同经历。

项目的核心目标是探索：一个 AI 如果要被用户感知为“认识我、懂我、和我有共同过去”，它的记忆系统应该如何组织。

## 项目定位

传统记忆系统通常把对话拆成事实条目，例如偏好、姓名、任务或历史问题。朋友类关系需要更复杂的记忆形态：某些时刻之所以重要，不是因为它包含明确事实，而是因为它承载了信任、脆弱、自我暴露、共同命名、未完成承诺或关系阶段变化。

Friend Memory Project 将这类长期关系记忆作为一个完整系统来设计。它试图把“朋友感”拆成可观察、可控制、可验证的机制，而不是只依赖模型临场生成亲密话术。

## 核心能力

- **关系视角的长期记忆**：记忆不只是用户事实，也包括双方关系如何发展、哪些互动改变了信任和亲密度。
- **共同经历沉淀**：把反复出现的事件、称呼、玩笑、约定和转折点组织成共同故事，而不是散落的聊天片段。
- **情绪和重要性判断**：区分普通信息、情绪时刻、脆弱表达、重大承诺和核心身份，让不同记忆有不同处理方式。
- **朋友式回忆**：在合适的时候提起旧事、延续未完成的话题、记得承诺和纪念日，但避免频繁打扰或机械复读。
- **关系阶段理解**：识别从陌生、试探、熟悉、亲密到冲突、疏远、修复等关系变化，让记忆使用符合当下关系状态。
- **用户控制权**：用户可以查看、修改、抑制、删除、重置或导出关系记忆，避免“被记住”变成不可控的负担。
- **安全边界**：对依赖风险、未成年人场景、危机信号和高敏信息保留额外保护，避免朋友感变成情感操控。
- **AI 参与可解释**：区分本地规则、外部模型判断和 fallback 情况，让系统知道一条记忆为什么被保存、如何被使用。

## 设计重点

这个项目强调“关系记忆”而不是“知识库记忆”。它关心的不是把所有信息永久存下来，而是判断什么值得被记住、什么应该逐渐淡化、什么必须由用户确认、什么需要在删除后彻底退出关系语境。

项目同时强调完整生命周期：记忆从产生、检索、巩固、降权、纠正、删除到审计，都应该有清晰规则。朋友类 AI 的长期体验不能只靠新增记忆，还需要遗忘、克制、边界和透明度。

## 适用场景

- 长期陪伴型 AI
- 朋友型或伙伴型智能体
- 个人助理的关系记忆层
- 情绪陪伴产品的记忆机制研究
- AI 角色长期一致性与共同经历建模
- 人机关系、记忆伦理和用户控制机制探索

## 项目价值

Friend Memory Project 提供的是一套“朋友类记忆系统应该具备什么”的工程化样本。它把抽象的朋友感拆成多个具体问题：如何识别重要时刻、如何形成共同故事、如何避免记忆伤害、如何处理拒绝和承诺、如何让用户知道 AI 到底记住了什么。

因此，这个项目既可以作为长期关系型 AI 的原型基础，也可以作为评估同类系统时的参考框架。
